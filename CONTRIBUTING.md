# Contributing to IronSights Vibe Security

Contributions are welcome and appreciated. This tool improves when the security community adds new patterns, CVEs, and real-world findings.

## What to contribute

- **New vulnerability patterns** — something AI coding tools introduce that isn't covered yet
- **Updated CVE references** — new CVEs affecting common vibe-coded stacks
- **New tool support** — patterns specific to Windsurf, Bolt, v0, or other AI coding tools
- **Better fixes** — clearer before/after code examples
- **New reference files** — a technology or framework not currently covered

## How to contribute

1. Fork the repo
2. Make your changes in the relevant reference file under `references/`, or add a new one
3. Update `skill.md` if you're adding a new audit category
4. Open a pull request with a short description of what you changed and why

## What makes a good contribution

- Based on a real vulnerability pattern — not theoretical
- Includes a concrete code example showing the vulnerability and the fix
- References a CVE, research paper, or documented breach where possible
- Stays focused on AI-generated code specifically — general security advice belongs elsewhere

## Questions?

Open an issue or reach out to [IronSights](https://ironsights.com.au/contact).
